v0.2.0 · Beta · MIT

7 tools exposés aux agents IA, validation Zod sur chaque input, resolution par FACEIT nickname ou Supabase user_id, RLS-protected. Les 3 tools v0.2 (coaching primitives) permettent à un agent de sortir du coaching CS2 niveau staff Vitality.

list_recent_demos
Historique des N dernieres demos d'un joueur (FV Rating, map, rounds, kills, date).
get_demo_analysis
Analyse complète d'une demo par UUID (FV Rating, scoreboard, statut parsing, lien partage).
get_player_stats
Stats agrégées sur N dernieres demos (FV moyen / min / max, tendance, maps préférées).
search_articles
Recherche dans le blog FragValue par mot-clé (titres, tags, keywords ponderes).
get_death_events NEW v0.2
Liste les morts d'un joueur dans une demo : round, time-in-round, position, killer, weapon, side T/CT, headshot. Resume early-deaths et HS rate.
get_round_breakdown NEW v0.2
Breakdown round-by-round : duree, first kill, winner, kills T vs CT, rôle du joueur cible. Tableau condense ou détail d'un round.
get_player_duels NEW v0.2
Matrice de duels : entries (win rate T/CT), trades in/out, HS rate, weapon usage, top opponents. Inclut insights brutes pour le coaching.

// PRIVACY & DATA

Privacy by default.

Aucune donnée user n'est exposée au MCP server par défaut. L'opt-in est explicite, granulaire et réversible.

Opt-in public_visibility

Tant qu'un user n'active pas public_visibility dans ses settings, ses profils et démos sont totalement invisibles au MCP server (Supabase RLS l'enforce au niveau DB, pas juste au niveau app).

Ce qui est exposé une fois opt-in : nickname FACEIT (déjà public sur FACEIT), map, rounds, total kills, FV Rating, date d'analyse. Jamais : email, IP, fichier de démo, trajectoires de grenades, conversations Coach IA.

  • Compte › Réglages › API & Integrations › Activer la visibilité publique
  • Le toggle peut être desactive a tout moment (RLS recheck en temps réel)
  • L'historique avant l'opt-in n'est jamais rendu public rétroactivement

// FAQ

Questions fréquentes.

Les 6 questions les plus posees pendant la beta privée.

C'est quoi un MCP server vs un API REST classique ?
MCP (Model Context Protocol) est un standard d'Anthropic pour exposer des tools a des agents IA. Au lieu qu'un dev humain écrive du code pour appeler ton API, c'est un agent IA (Claude, ChatGPT) qui découvre et appelle les tools tout seul. L'API REST reste utile pour les apps custom (Discord bots, mobile apps). Le MCP est spécifique aux agents IA. Les deux sont complémentaires — FragValue va shipper les deux d'ici Q3 2026.
Faut-il être user Pro FragValue pour utiliser le MCP server ?
Non pour la v0.1 read-only. Tu peux interroger les profiles publics et leurs demos complètes sans compte FragValue. Pour les actions write (analyser une nouvelle demo) en v0.3, il faudra être Pro avec un OAuth flow. Après Q1 2027, des tiers de pricing s'appliqueront sur les volumes d'API calls (Free / Pro / Enterprise).
Comment FragValue protégé les données user ?
Privacy by default. Aucune donnée user n'est exposée par défaut. L'user doit activer explicitement public_visibility dans ses settings pour que son profile et ses demos complètes soient queryables via MCP. Le Row Level Security (RLS) Supabase enforce cette regle au niveau DB, pas juste au niveau app. Voir la section "Privacy & Data" ci-dessus pour le détail technique.
Combien ça coute ?
Gratuit pendant la beta (jusqu'à Q1 2027). Après, 3 tiers sont prevus : Free 100 requêtes par mois pour les hobbyistes, Pro 10k par mois inclus dans l'abonnement FragValue Pro 9 EUR/mois, Enterprise avec quotas custom + SLA pour les orgs esports / studios. Toutes les requêtes faites pendant la beta ne sont jamais facturees retroactivement.
Quels agents supportent MCP aujourd'hui ?
Claude Desktop (Anthropic), Cursor (l'IDE IA), Cline (extension VS Code), ChatGPT custom GPTs. Plus de 50 clients sont listés sur la registry officielle Anthropic. MCP est le standard ouvert le plus rapide a croître en 2026 — Google, Microsoft et OpenAI ont annonce des compatibilities natives. En clair : ton tool est dispo partout en branchant 1 fichier de config.
Que faire si un tool plante ou renvoie une erreur ?
Le MCP server retourne un message d'erreur structuré (McpError). L'agent IA peut le voir et te l'afficher, ou retry automatiquement. En cas de bug repete, ouvre une issue sur github.com/Ladreuille/fragvalue-mcp/issues. Le repo etant open source, tu peux aussi PR un fix.
